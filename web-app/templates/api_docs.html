IPO Pilot API Documentation

All API endpoints require authentication with a valid JWT token.

Authentication

Include your JWT token in the Authorization header:

Authorization: Bearer YOUR_JWT_TOKEN

Endpoints

GET /api/ipos/live

Get all currently open IPOs

Returns: List of IPO objects

GET /api/ipos/upcoming

Get upcoming IPOs

Returns: List of upcoming IPO objects

POST /api/monitor/start

Start monitoring an IPO

Body: { profile_id: number, ipo_id: number }

POST /api/monitor/stop

Stop monitoring an IPO

Body: { session_id: number }

GET /api/monitor/status

Get monitoring status

Returns: Array of active monitoring sessions

POST /dashboard/apply/:ipo_id

Apply to an IPO

Body: { profile_id: number, kittas: number }

Example Response

{
  "ipos": [
    {
      "id": 1,
      "company_name": "Nabil Bank",
      "share_id": "NABIL",
      "open_date": "2026-02-08",
      "close_date": "2026-02-15",
      "total_kittas": 500000,
      "price_per_unit": 1000
    }
  ]
}

Error Codes

Code Meaning
400 Bad Request - Invalid parameters
401 Unauthorized - Missing or invalid token
403 Forbidden - No active subscription
404 Not Found - Resource not found
500 Internal Server Error

Rate Limiting

API requests are rate limited to 100 requests per minute per user.
